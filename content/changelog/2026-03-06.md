## 2FA

Neon now supports **two-factor authentication (2FA)** for your account. Once enabled, you enter a 6-digit code from your authenticator app (for example, Google Authenticator, Authy, or 1Password) each time you log in. Set it up from your profile menu: **Account settings** → **Set up two-factor authentication**, then scan the QR code and verify with a code from your app. Removal is delayed by 24 hours for security. For details, see [Two-factor authentication](/docs/manage/accounts#two-factor-authentication).

## Collaborate on the free tier

You can now collaborate with teammates on the free tier without a credit card. Invite others to your org, or keep separate workspaces for personal projects, side projects, and open source. Free plan limits have been updated. You can create **multiple free organizations** (up to 1,000 per account) and each free organization can have **up to 1,000 members**. The only limit is the 1,000 free organizations per account; if you reach that, you'll see an error that you've reached the maximum number of free organizations. For details, see [Manage billing](/docs/manage/orgs-manage#manage-billing) and [Neon plans](/docs/introduction/plans).

<Admonition type="info" title="What are orgs and members?">
An **organization** is a workspace that owns projects. **Members** are people you invite to the organization (as Admin or Member); they can access the org's projects and, for Admins, manage the org and invite others. Free plan orgs are for personal or small-team use; you can now have multiple free orgs and more members per org on the free tier.
</Admonition>

## Data API Advisors

**Data API Advisors** is a new tab on the **Monitoring** page that helps you secure and tune your database when you use the [Neon Data API](/docs/data-api/overview), Neon's HTTP interface for querying your database from browsers, serverless functions, and edge runtimes. Because the Data API exposes your schema directly over HTTP, the RLS policies and security you define on your schema are critical. The advisors scan your database and report security and performance issues (such as missing RLS, sensitive columns exposed, or unindexed foreign keys) with severity levels and recommended fixes.

In the Neon Console, go to **Monitoring > Data API Advisors** for your branch to run a scan and view issues grouped by category.

![Data API advisor](/docs/data-api/data-api-database-advisor-monitor.png)

For details, see [Data API Advisors](/docs/data-api/database-advisor).

## Organization settings for Neon Auth

You can now configure **Organizations** for Neon Auth from the Neon Console. In your project, open **Auth** in the sidebar and go to the **Configuration** tab to find the new Organizations section. You can enable or disable organizations, set a maximum number of organizations per user, allow or disallow users to create organizations, and choose the default role for organization creators (admin or owner). These settings support multi-tenant apps where users create and join organizations. For details, see [Organization](/docs/auth/guides/plugins/organization) in the Neon Auth docs.
