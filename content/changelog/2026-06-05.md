## Link, branch, and query from the Neon CLI

**neonctl 2.22.2** is a major CLI update that brings Vercel-style project linking, branch checkout, a top-level `psql` command, and full Data API management to the terminal. If you use the [Neon CLI](/docs/reference/neon-cli), upgrade to access these commands.

**New commands**

- [`neon link`](/docs/reference/cli-link) — Bind the current directory to a Neon project. Writes a `.neon` context file with your `orgId`, `projectId`, and `branchId`. Supports interactive prompts, non-interactive flags (`--org-id`, `--project-id`, `--params`), and `--agent` JSON mode for AI coding assistants.
- [`neon checkout`](/docs/reference/cli-checkout) — Pin a branch in `.neon` so subsequent commands target it without passing a branch on every command. New projects create a default branch named `production`.
- [`neon psql`](/docs/reference/cli-psql) — Connect to a database via `psql` as a dedicated top-level command (equivalent to `neon connection-string --psql`).
- [`neon data-api`](/docs/reference/cli-data-api) — Provision and manage the [Neon Data API](/docs/data-api/overview) with `create`, `get`, `update`, `refresh-schema`, and `delete` subcommands.

**[`neon set-context`](/docs/reference/cli-set-context) improvements**

- **`--branch-id`** — Set which branch your commands target, the same way you set `--project-id` and `--org-id`.
- **Find your project from subfolders** — Run CLI commands from any subdirectory. The CLI walks up parent folders looking for a `.neon` file to load your linked project.
- **Auto-add `.neon` to `.gitignore`** — The first time a `.neon` file is created, the CLI adds it to `.gitignore` in that folder so local project settings are not committed by accident.

A typical workflow:

```bash
neon link
neon checkout production
neon psql
```

For command reference and options, see the [Neon CLI overview](/docs/reference/neon-cli) or get set up quickly with the [CLI Quickstart](/docs/reference/cli-quickstart).

**Upgrade to neonctl 2.22.2**

These commands require **neonctl 2.22.2** or later. Check your installed version:

```bash
neon --version
```

To upgrade, use the method that matches how you installed the CLI:

```shell
# npm
npm update -g neonctl

# Homebrew
brew upgrade neonctl
```

For full upgrade instructions (including binary installs), see [Neon CLI upgrade](/docs/reference/cli-install#upgrade).

If you use the Neon CLI in CI/CD tools like GitHub Actions, you can safely pin it to `@latest`, as we prioritize stability for CI/CD processes. For example:

```yaml
- name: Install Neon CLI
  run: npm install -g neonctl@latest
```

## 5x more data transfer on all paid plans

We've increased the data transfer (egress) allowance on all paid plans from 100 GB to 500 GB per month. The new allowance takes effect automatically with no changes required on your end, and it will be reflected on your June invoice.

## Faster Text-to-SQL in the SQL Editor

Text-to-SQL suggestions in the Neon SQL Editor are now significantly faster. You'll notice quicker responses when asking the AI to generate queries from natural language, so there's less waiting and more querying.

## Postgres 18 for newly created Neon projects

![Create PG18 project](/docs/changelog/create_project_18.png)

Postgres 18 is now the default for newly created Neon projects. Neon continues to support Postgres 14, 15, 16, and 17, if you prefer to stick with those. For Neon's Postgres version support policy, see [Postgres Version Support](/docs/postgresql/postgres-version-policy).

## Expanded infrastructure capacity in AWS Europe (Frankfurt)

We've expanded infrastructure capacity in the AWS Europe (Frankfurt) region (`eu-central-1`) with new NAT gateway IP addresses and new VPC endpoint service addresses for Private Networking.

<Admonition type="tip" title="Update your IP allowlists">
If you have IP allowlists on external systems that Neon connects to, **update those allowlists to include the new NAT gateway addresses**. Connections may be affected intermittently if traffic routes through non-allowlisted NAT gateways.
</Admonition>

If you use Private Networking in `eu-central-1`, you can now use the additional VPC endpoint service addresses for enhanced capacity and reliability. See the [Regions documentation](/docs/introduction/regions#aws-nat-gateway-ip-addresses) for the complete list of NAT gateway IPs and the [Private Networking guide](/docs/guides/neon-private-networking) for VPC endpoint service addresses by region.
