## Unlimited orgs and members on the Free Plan

You can now create **unlimited free organizations** per account and add **unlimited members** to each on the Free plan. Collaborate with teammates, invite others to your org, or keep separate workspaces for personal projects, side projects, and collaborations. No credit card required. For details, see [Manage billing](/docs/manage/orgs-manage#manage-billing) and [Neon plans](/docs/introduction/plans).

![Create organization menu](/docs/changelog/free_orgs.png)

<Admonition type="info" title="What are orgs and members?">
An **organization** is a workspace that owns Neon projects. **Members** are people you invite to the organization. [Learn more](/docs/manage/orgs-manage).
</Admonition>

## Organizations API

The API for listing organization members now supports **pagination** and **sorting**. You can use `limit` (1–500, default 500) and `cursor` to page through members, and `sort_by` (`email`, `role`, or `joined_at`) and `sort_order` to control order. The default limit is 500, so existing integrations continue to work without changes. Member objects now include an optional **`has_mfa`** field indicating whether the member has TOTP (2FA) enabled. In the Console, the Organization members page shows a 2FA indicator for each member and supports sorting and pagination.

## Monitoring network transfer

**Network transfer** (egress) is data sent from your Neon databases to clients. It is one of the usage metrics that affects your bill on paid plans. Many teams only notice it when it shows up as a line item, so we want to make it easier to understand and stay on top of. We've added a guide that explains what network transfer is, what typically drives it (queries, pg_dump, logical replication, log export), how to monitor it in the Console and via the Consumption API, and how to manage or reduce usage. See [Reduce network transfer costs](/docs/introduction/network-transfer).

## New NAT gateway IP addresses

We've added new NAT gateway IP addresses in the AWS US East (N. Virginia) region to expand infrastructure capacity. If you have IP allowlists on external systems that Neon connects to, **update those allowlists to include the new addresses**. Connections may be affected intermittently if traffic routes through non-allowlisted NAT gateways.

See our [Regions documentation](/docs/introduction/regions#aws-nat-gateway-ip-addresses) for the complete list of NAT gateway IPs for all regions.

## New VPC endpoint services for Private Networking

We've added new VPC endpoint service addresses for Private Networking in the AWS US East (N. Virginia) region. If you've set up Private Networking in `us-east-1`, you can now use the additional endpoint service addresses for enhanced infrastructure capacity and reliability.

For the complete list of VPC endpoint service addresses by region, see our [Private Networking guide](/docs/guides/neon-private-networking).

## Organization settings for Neon Auth

You can now configure **Organizations** for Neon Auth from the Neon Console. Go to **Auth** > **Configuration** > **Organizations** (per branch) to enable or disable the plugin, set the maximum organization memberships per user and the maximum members per organization, choose the creator role (owner or admin) for new organizations, and control whether invitation emails are sent. These settings support multi-tenant apps where users create and join organizations. For details, see [Organization](/docs/auth/guides/plugins/organization) in the Neon Auth docs.

## Neon Auth webhooks

Introducing **Neon Auth webhooks**. You can configure webhooks to receive HTTP POST requests when authentication events occur (OTP delivery, magic link delivery, user creation). Use them to replace built-in email delivery with your own channels (SMS, custom email, WhatsApp), validate signups before they complete, or sync new users to CRMs and analytics. The guide covers event types, API configuration, payload structure, signature verification, expected responses, retry behavior, and testing. See [Webhooks](/docs/auth/guides/webhooks).
