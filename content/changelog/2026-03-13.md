## Organizations API

The API for listing organization members now supports **pagination** and **sorting**. You can use `limit` (1–500, default 500) and `cursor` to page through members, and `sort_by` (`email`, `role`, or `joined_at`) and `sort_order` to control order. The default limit is 500, so existing integrations continue to work without changes. Member objects now include an optional **`has_mfa`** field indicating whether the member has TOTP (2FA) enabled. In the Console, the Organization members page shows a 2FA indicator for each member and supports sorting and pagination.

## Neon Auth webhooks

Introducing **Neon Auth webhooks**. You can configure webhooks to receive HTTP POST requests when authentication events occur (OTP delivery, magic link delivery, user creation). Use them to replace built-in email delivery with your own channels (SMS, custom email, WhatsApp), validate signups before they complete, or sync new users to CRMs and analytics. The guide covers event types, API configuration, payload structure, signature verification, expected responses, retry behavior, and testing. See [Webhooks](/docs/auth/guides/webhooks).
