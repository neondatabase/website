## Faster writes on Neon

Neon now delivers significantly faster writes across all projects. For write-heavy workloads, this optimization delivers up to a **5x performance improvement** in our testing.

![HammerDB benchmark](/docs/changelog/hammerdb_throughput.jpg)

This performance enhancement comes from reducing extra write overhead in Postgres and moving more of that work to Neon's storage layer, while preserving durability.

For a technical deep dive, see the blog post: [Everyone gets faster writes: Turning off FPW on Neon](https://neon.com/blog/turning-off-fpw-for-faster-writes).

## Snapshot billing

**Snapshot storage pricing took effect on May 1, 2026**. Snapshots are now billed at $0.09/GB-month. This applies to both manual snapshots and snapshots created by automated backup schedules.

To review snapshot billing, visit the **Billing** page in the Neon Console. To view snapshot storage, see the **Backup & Restore** page. For details, see [Backup & restore](/docs/guides/backup-restore).

![Snapshot storage size](/docs/changelog/snapshot_storage_size.png)

## Require two-factor authentication for your organization

Organization **admins** can now require **two-factor authentication** for everyone in the organization from **Organization → Settings**. You can only enable this if **your own account already uses 2FA**, so you don't lock yourself out while rolling out the policy.

![Require 2FA for orgs](/docs/changelog/require_org_2fa.png)

When the org requires 2FA and a member still needs to enroll, the Neon Console shows **Set up two-factor authentication**: scan the **QR code** with an authenticator app or **enter the manual key**, then type the **6-digit code** and choose **Verify and enable**. Recovery codes are not available yet. Keep your authenticator backed up or synced across devices. If you lose access, contact [Support](/docs/introduction/support).

For optional 2FA on your Neon account (outside of org enforcement), see [Manage your Neon account](/docs/manage/accounts).

## Fixes & improvements

<details>
<summary>**Branch expiration support for Neon Auth**</summary>

Fixed an issue where enabling [Neon Auth](/docs/auth) on a branch could block branch expiration (TTL) updates. You can now set or change expiration dates for Neon Auth branches. See [Branch expiration](/docs/guides/branch-expiration).

</details>

<details>
<summary>**Snapshot restore improvement**</summary>

Fixed an issue where finalizing a snapshot restore could leave snapshot schedules attached to the renamed old branch. Snapshot schedules now move to the finalized branch in both restore flows, so scheduled backups continue on the active branch after finalize.

</details>

<details>
<summary>**Easily view deactivated organization members**</summary>

[Retrieve organization members](https://api-docs.neon.tech/reference/getorganizationmembers) (`GET /organizations/{org_id}/members` on `https://console.neon.tech/api/v2`) now includes optional `deactivated_at` on each member's `user` object when that account is deactivated. In **Organization → People**, the Console shows a **Deactivated** badge in the member list (not only on the user profile), so access reviews no longer require opening every row.

</details>
