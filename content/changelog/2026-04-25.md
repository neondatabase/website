## Spending alerts

Organization admins can now get **spending alerts** when monthly charges approach or reach a [spending limit](/docs/introduction/spending-limit). Neon emails you at **80%** and **100%** of your spend limit. Neon checks spend about every **15 minutes** and sends weekly reminders until you raise the limit or the billing cycle resets. You set and manage limits on the **Billing** page in the Neon Console.

For more information, see **[Spending alerts](/docs/introduction/spending-limit)**.

## Project consumption API: snapshot storage metric

The [Retrieve project consumption metrics](https://api-docs.neon.tech/reference/getconsumptionhistoryperprojectv2) endpoint (`GET /api/v2/consumption_history/v2/projects`) now supports a **`snapshot_storage_bytes_month`** in the `metrics` parameter so you can track monthly snapshot storage. Snapshot storage billing starts **May 1, 2026**; until then snapshots remain free during the beta period. See [Backup & restore](/docs/guides/backup-restore) and [Plans](/docs/introduction/plans) for how snapshot storage is priced.

For the full metrics table, parameters, and examples, see [Querying consumption metrics](/docs/guides/consumption-metrics).

## Durable AI agents with Pydantic AI, DBOS, and Neon

Multi-step agents depend on LLMs and tools that can time out, rate limit, or fail mid-run. A practical pattern is **durable execution**: checkpoint progress in **Postgres** so a workflow can resume after a crash or retry without redoing expensive steps from scratch. **DBOS** provides that durable execution layer, **Pydantic AI** structures agent logic and tool orchestration, and **Neon** backs the database so checkpoint state lives in serverless Postgres that scales with your workload.

For a full walkthrough with a working example, see [**Building Durable AI Agents with Pydantic AI, DBOS, and Neon**](/guides/pydantic-ai-dbos-neon).

## Optimizing Neon docs for AI agents

We've been making our docs more useful to AI agents. [**Agents grew up, so did our docs**](https://neon.com/blog/agents-grew-up-so-did-our-docs) describes what we put in place: MDX converted to Markdown at build time, middleware that serves Markdown for agent-shaped requests, the same doc paths with a `.md` suffix, and a structured [`llms.txt`](https://neon.com/docs/llms.txt) index.
