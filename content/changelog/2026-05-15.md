## Neon Auth

Three new updates to Neon Auth this week:

**Magic link sign-in** — Neon Auth now supports the Magic link plugin for passwordless sign-in: users receive an email with a link to authenticate without a password. Enable the plugin from the Neon Console (**Auth** > **Plugins**) or via the Neon API, then use the Neon Auth SDK and UI components in your app. See [Magic link plugin](/docs/auth/guides/plugins/magic-link) for setup details.

**Phone number sign-in** — The new Phone Number plugin lets users sign in with a one-time password (OTP) delivered by SMS. You bring your own SMS provider (Twilio, MessageBird, Vonage, and others) and connect it via a `send.otp` webhook. Enable the plugin in the Console or via the API, and handle delivery through the new `phone_number.verified` webhook event. See [Phone number plugin](/docs/auth/guides/plugins/phone-number) for setup details.

**Custom application name** — You can now set a custom application name that appears in user-facing messages like email verification and password reset emails. Set it in the Console under your auth settings or via the API using `PATCH /auth/config`. The name is configured per branch, so you can use different values for development and production. See [Production checklist](/docs/auth/production-checklist) for details.
