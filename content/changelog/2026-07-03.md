## Recover a deleted branch

Branches are now soft-deleted by default. When you delete a branch, it enters a 7-day deletion recovery period before it's permanently removed, giving you time to undo an accidental deletion. Recovering a branch restores its data and configuration, returns endpoints to an idle state, and keeps existing connection strings valid. You can list recoverable branches with `include_deleted=true` and restore one with the new recover endpoint. If you need to bypass the recovery window, pass `hard_delete=true` on the delete request. To learn more, see [Recover a deleted branch](/docs/manage/branches#recover-a-deleted-branch).
